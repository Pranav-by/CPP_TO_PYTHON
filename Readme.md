## ğŸ§  How It Works (Compiler Flow)

### ğŸ§© 1. Lexer (`lexer.cpp`)

* Reads the source code character by character.
* Groups them into **tokens** like identifiers, numbers, operators, etc.
* Example:

  ```cpp
  int x = 10;
  ```

  becomes tokens:
  `[KEYWORD_INT] [IDENTIFIER:x] [EQUALS] [NUMBER_LITERAL:10] [SEMICOLON]`

---

### ğŸ§© 2. Parser (`parser.cpp`)

* Uses **recursive descent parsing** to create an **Abstract Syntax Tree (AST)**.
* Example:

  ```cpp
  int x = 10;
  ```

  becomes:

  ```
  VarDecl(Type: int, Name: x, Value: Number(10))
  ```

---

### ğŸ§© 3. Code Generator (`codegen.cpp`)

* Walks through the AST and generates equivalent **Python syntax**.
* Handles indentation, loops, and print formatting.
* Example:

  ```
  VarDecl(Type: int, Name: x, Value: Number(10))
  ```

  â†’ becomes:

  ```python
  x = 10
  ```
